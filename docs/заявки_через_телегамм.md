# Заявки на закупку через Telegram

**Цепочка процесса:**
1. **Сотрудник** (Role: employee) → Создает заявку (товар, кол-во) → Статус `pending_approval`.
2. **Ответственный** (Role: manager) → Согласовывает заявку → Статус `approved` → Уведомление складу.
3. **Склад** (Role: warehouse) → Загружает счет на оплату → Статус `invoice_uploaded` → Уведомление бухгалтерии.
4. **Бухгалтерия** (Role: accountant) → Берет в работу (`processing`) → Подтверждает к оплате → Статус `awaiting_payment`.
5. **Босс** (Role: boss) → Оплачивает счет → Статус `paid` → Уведомление всем участникам.

**Структура данных:**
- Таблицы: `telegram_bot_users` (роли), `procurement_requests` (заявки), `procurement_history` (лог).
- Статусы: `draft` → `pending_approval` → `approved` → `invoice_uploaded` → `processing` → `awaiting_payment` → `paid` (или `rejected` на любом этапе).
