# LC / ДС rollout (Stage 1-2)

**Дата:** 08.03.2026  
**Статус:** In progress  
**Цель:** добавить parallel flow для LC / Дополнительных соглашений без замены текущей логики `estimates`, `work_items`, `VOR`.

## Что изменено

### БД
- Добавлена миграция `supabase/migrations/20260308123000_add_estimate_revisions_and_addendum_flow.sql`
- Добавлена колонка `public.estimates.position_id`
- Добавлена колонка `public.vors.baseline_revision_id`
- Добавлена таблица `public.estimate_revisions`
- Добавлена таблица `public.estimate_revision_items`
- Добавлены индексы для revision-flow
- Добавлены RLS policy для новых таблиц

### Flutter / Dart
- Добавлена доменная сущность `lib/domain/entities/estimate_revision.dart`
- Расширен `EstimateRepository` новым API для LC / ДС
- Расширен `SupabaseEstimateDataSource` новым isolated flow
- Добавлен Excel-сервис `lib/features/estimates/presentation/services/estimate_addendum_excel_service.dart`
- Добавлено новое modal-окно `lib/features/estimates/presentation/screens/import_estimate_addendum_modal.dart`
- В `EstimateDesktopView` добавлена кнопка `LC / ДС`

## Что важно

- Старый импорт сметы через `ImportEstimateFormModal` не заменён.
- Старые запросы в `estimates` не переписаны на revision-flow.
- `populate_vor_items()` пока не использует `estimate_revisions`.
- Текущий ВОР flow должен продолжать работать как раньше.
- Новый поток LC / ДС пока создаёт только `draft`-ревизии в новых таблицах.

## Что делает новый поток сейчас

1. Пользователь открывает смету, привязанную к договору.
2. Нажимает `LC / ДС`.
3. Скачивает Excel текущей сметы с колонкой `ID позиции`.
4. Меняет существующие строки или добавляет новые.
5. Загружает Excel обратно.
6. Приложение:
   - при необходимости создаёт базовую ревизию `Основная`
   - создаёт новую `draft`-ревизию `ДС-*`
   - сохраняет строки в `estimate_revision_items`
   - отсутствующие строки помечает как `removed`

## Что пока НЕ подключено

- Переключение активной сметы на `estimate_revisions`
- Привязка новых ВОР к `baseline_revision_id`
- Расчёт превышения относительно `ДС-*`
- Обратная синхронизация `draft`-ревизии в `estimates`
- UI просмотра истории ревизий

## Ручная проверка после выката

1. Открыть существующий список смет.
2. Открыть карточку старой сметы.
3. Проверить, что старый импорт сметы работает без изменений.
4. Проверить, что создание ВОР работает без изменений.
5. Создать новый договор и загрузить обычную смету старым способом.
6. Открыть эту смету и нажать `LC / ДС`.
7. Скачать Excel, убедиться, что есть колонка `ID позиции`.
8. Изменить одну существующую строку и добавить одну новую строку без `ID позиции`.
9. Загрузить файл обратно.
10. Проверить в БД:
    - есть запись в `estimate_revisions`
    - есть строки в `estimate_revision_items`
    - базовая ревизия `Основная` создана автоматически при первом запуске

## При нештатном результате

- Не удалять старые данные в `estimates`
- Не менять старый импорт сметы
- Не подключать `estimate_revisions` к ВОР до завершения проверки
- Сначала проверить:
  - наличие миграции в БД
  - RLS policy
  - наличие bucket `estimates`
  - ошибки в логах Flutter / Supabase
