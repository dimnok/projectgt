# Структура интерфейса модуля ФОТ

**Дата:** 26 мая 2026 года

## 📱 Основной экран (`PayrollListScreen`)
Экран построен на базе `CupertinoSlidingSegmentedControl` для переключения между 4 функциональными областями.

### 1. Вкладка "ФОТ"
Основная расчетная ведомость.
- **Виджет:** `PayrollTableWidget` (обертка) + `PayrollTableView` (реализация).
- **Desktop/Tablet:** `GTAdaptiveTable`. Колонки: Сотрудник, Часы, Ставка, База, Премии, Штрафы, Суточные, К выплате, Выплаты, Остаток, Баланс.
- **Mobile:** `PayrollMobileView` + `PayrollCard`. 
    - **Карточка сотрудника:** В заголовке ФИО (14pt), статус и сумма "К выплате" (14pt). Остальные детали (часы, ставки, развертка премий) скрыты в раскрывающемся блоке.
    - **Без контейнеров:** В мобильной версии карточки отображаются без внешних `Card`-оберток для экономии места.
    - **Нижняя панель (Sticky Bar):** Компактные общие итоги по списку (К выплате, Выплачено, Баланс).
- **Особенности:** 
    - **Адаптивность:** Полностью динамический расчет ширины колонок на основе контента.
    - **Минимализм:** Удалены должности сотрудников для упрощения восприятия данных.
    - **ФИО и Статус:** Отображается полное имя сотрудника (Фамилия Имя Отчество). Рядом с ФИО выводится текущий статус (работает, уволен, отпуск и т.д.) уменьшенным шрифтом (10pt) с цветовой индикацией согласно `EmployeeUIUtils`.
    - **Состав списка:** Отображаются все работавшие в месяце сотрудники + все сотрудники с ненулевым балансом (должники).
    - **Поиск:** При поиске по ФИО фильтруются как записи ФОТ, так и список сотрудников (включая тех, кто отображается только по балансу).
    - **Индикация:** Колонка "Баланс" содержит иконки направления (долг/переплата) и цветовое кодирование.
    - **Итоги:** Автоматический расчет строки "ИТОГО", которая динамически обновляется при поиске.
    - **Контекстное меню:** При нажатии на строку сотрудника открывается меню действий (Премия, Штраф, Выплата, Детали). Элемент "Детали" на данный момент находится в разработке.

### 2. Вкладка "Премии"
- **Виджет:** `PayrollTabBonuses` (экран) → `PayrollBonusTableWidget` (провайдер) → `PayrollBonusTableView` (реализация) → `FOTTransactionTable` (shared).
- **Функционал:** Отображение списка премий за выбранный месяц.
- **Колонки:** Дата, Сотрудник, Сумма, Объект, Примечание.
- **Данные:** Таблица `payroll_bonus`.
- **Особенности:** 
    - Использует унифицированный виджет `FOTTransactionTable` на базе `GTAdaptiveTable`.
    - **Итоги:** Добавлена строка «ИТОГО» с суммированием по отфильтрованному списку.
    - **Empty State:** Информативный блок при отсутствии записей с иконкой и подсказкой.
- **Управление:** Кнопка "Добавить" вынесена в заголовок таблицы. Редактирование и удаление доступны через контекстное меню строки.

### 3. Вкладка "Штрафы"
- **Виджет:** `PayrollTabPenalties` (экран) → `PayrollPenaltyTableWidget` (провайдер) → `PayrollPenaltyTableView` (реализация) → `FOTTransactionTable` (shared).
- **Функционал:** Отображение списка штрафов.
- **Колонки:** Дата, Сотрудник, Сумма, Объект, Примечание.
- **Особенности:** 
    - Унифицирован с таблицей премий через `FOTTransactionTable`.
    - **Итоги:** Добавлена строка «ИТОГО».
    - **Empty State:** Информативный блок при отсутствии записей.
- **Управление:** Кнопка "Добавить" в заголовке таблицы. Полное управление через контекстное меню.

### 4. Вкладка "Выплаты"
- **Виджет:** `PayrollTabPayouts` (экран) → `PayrollPayoutTableWidget` (провайдер) → `PayrollPayoutTableView` (реализация) → `FOTPayoutTable` (shared).
- **Функционал:** Просмотр истории выплат; ручное и массовое создание.
- **Колонки:** Дата, Сотрудник, Сумма, Способ, Тип, Комментарий.
- **Особенности:** 
    - Унифицирован через `FOTPayoutTable`.
    - **Итоги:** Добавлена строка «ИТОГО».
    - **Empty State:** Информативный блок при отсутствии записей.
- **Управление (заголовок таблицы, `PermissionGuard` payroll/create):**
    - **«Импорт из Excel»** — `PayrollPayoutExcelImportDialog` → выбор `.xlsx` → `PayrollPayoutImportPreviewDialog`.
    - **«Добавить»** — `PayrollPayoutFormModal` → `PayrollPayoutAmountModal` (ручной массовый ввод).
    - Редактирование и удаление — контекстное меню строки.

#### Импорт из Excel (26.05.2026)
| Шаг | Экран | Действие |
|-----|--------|----------|
| 1 | `PayrollPayoutExcelImportDialog` | Дата, способ, тип, комментарий → «Выбрать файл» |
| 2 | `PayrollPayoutImportPreviewDialog` | Таблица: Стр. · Статус · ФИО из файла · Сумма (на всю ширину окна) |
| 3 | Подтверждение | При расхождениях ФИО — диалог «Импортировать только найденных» |
| 4 | БД | Записи в `payroll_payout`; файл нигде не сохраняется |

**Статусы предпросмотра:** `Найден` (зелёный) · `Не найден` · `Неоднозначно`.

**Формат файла:** ведомость банка/1С — колонки с «ФИО» и «Сумма»; ФИО в порядке **Фамилия Имя Отчество**; уволенные сотрудники участвуют в поиске.

## 🔍 Поиск и фильтрация
- **Поиск:** Глобальный поиск по ФИО сотрудника (`PayrollSearchAction`).
- **Фильтр периода:** Выбор месяца и года (`PayrollFilterWidget`).

## 📥 Экспорт и импорт

| Операция | Где в UI | Механизм |
|----------|----------|----------|
| **Экспорт ведомости ФОТ** | AppBar основного экрана (`PayrollExportAction`) | Edge Function `export-payroll` (сервер, ExcelJS, данные FIFO) |
| **Импорт выплат** | Вкладка «Выплаты» → «Импорт из Excel» | Клиент: `file_picker` + `excel`, без Storage и без Edge Function |

Шаблон Excel для импорта **не** предусмотрен — используется файл из банка в текущем виде.
