# Модуль Договоры (Contracts)

**Дата актуализации:** 1 марта 2026 года
**Изменения:** 
- Внедрена система **Global On Focus Refresh (TTL 5 минут)**. Договоры автоматически обновляются в фоне при возврате в приложение.
- Устранено обрезание номера договора в десктопном реестре за счет оптимизации распределения ширины колонок.
- Удалено избыточное контекстное меню в списке договоров; управление теперь осуществляется через панель деталей.
- Реализована плавная анимация появления окна деталей (`fade` + `scale`) и каскадная анимация элементов внутри панели с использованием `flutter_animate`.
- Полный редизайн десктопного интерфейса списка договоров (карточный вид вместо таблицы).
- Добавление отображения Объекта и счетчика дней до окончания договора в списке.
- Рефакторинг форм создания/редактирования с использованием дизайн-системы GT (GTTextField, GTDropdown, GTButtons).
- Актуализация структуры файлов и БД (Audit).

## ⚠️ Важное замечание
Модуль является владельцем таблицы `contracts` и связанных с ней `contract_files`, `ks2_acts` и `estimates`. Все данные жестко изолированы на уровне БД через `company_id`. Модуль критически зависит от справочников `contractors` и `objects`. Подключен к глобальному координатору обновления данных.

## 📝 Описание
Модуль **Contracts** предназначен для управления юридическими и финансовыми договорами с заказчиками и подрядчиками. Он обеспечивает полный жизненный цикл договора: от регистрации и загрузки файлов до контроля выработки (выполненных работ) и формирования актов КС-2.

**Ключевые функции:**
- **Global On Focus Refresh:** Автоматическое обновление списка договоров при возврате в приложение (TTL 5 мин) без прерывания работы пользователя (Quiet Refresh).
- Регистрация договоров с привязкой к Объекту и Контрагенту.
- Учет финансовых параметров: НДС, авансы, гарантийные удержания, генподрядный процент.
- Визуальный контроль сроков: автоматический расчет дней до завершения с цветовой индикацией.
- Хранение сканов и документов договора (`contract_files`).
- Ведение сметной документации (`estimates`).
- Расчет оперативной выработки на основе закрытых смен (`ContractWorksService`).
- Формирование и учет актов выполненных работ КС-2 (`ks2_acts`).

## 🔗 Зависимости
- **Владение (Owner):** `contracts`, `contract_files`, `ks2_acts`, `estimates`.
- **Использование (Usage):** `contractors` (Контрагенты), `objects` (Объекты), `companies` (Ядро), `profiles` (Создатели записей), `work_items` (для расчета выработки).
- **Infrastructure:** `AppFocusRefreshCoordinator` для автоматического обновления.

## 🖼️ Слой Presentation
### Экраны
- `ContractsListScreen`: Главный экран со списком, поиском и фильтрацией. Точка регистрации `RefreshTarget` ('contracts'). Адаптивно переключается между Mobile и Desktop представлениями.
- `ContractsListDesktopView`: Полноэкранный реестр договоров. Заголовок включает поиск по номеру, контрагенту или объекту.
- `ContractsListMobileView`: Оптимизированный список для мобильных устройств.
- `ContractFormScreen`: Адаптивная форма создания/редактирования договора. Использует `GTTextField`, `GTDropdown` и `GTButtons` для единообразия UI.

### Виджеты
- `AppBarWidget`: Отображает индикатор фонового обновления при работе координатора.
- `ContractTableView`: Основной виджет списка на Desktop. Реализован как `ListView.separated` из карточек `_ContractCard`. Оптимизирован `flex` колонок для корректного отображения длинных номеров.
- `_ContractCard`: Строка-карточка договора. Отображает ключевые поля: Номер, Контрагент, Объект, Дата окончания (со счетчиком дней), Сумма, Статус. Взаимодействие осуществляется через одиночный тап (открытие деталей).
- `ContractDetailsPanel`: Панель с детальной информацией. Использует `showGeneralDialog` для плавного анимированного появления и `flutter_animate` для внутреннего каскадного отображения элементов.

### Провайдеры
- `contractProvider`: Состояние списка и CRUD операции. Поддерживает `loadContracts(quiet: true)` для фонового обновления без Loading Spinner.
- `contractWorksSummaryProvider`: Асинхронный расчет выработки по договору.
- `contractFilesProvider`: Состояние файлов текущего договора.

## ⚙️ Слой Domain/Data
- **Сущности:** `Contract`, `ContractFile`, `Ks2Act`, `Estimate` (Freezed, включают `companyId`).
- **Модели:** `ContractModel` (с расширенными полями подписантов и финансовых условий).
- **Репозитории:** `ContractRepository` (Supabase).

## 🗄️ База данных (Audit)

### Таблицы
1. **`contracts`**: Основные данные (номер, дата, сумма, реквизиты, подписанты).
   - `company_id` (UUID, NOT NULL) — FK на `companies`.
   - `object_id` (UUID) — FK на `objects`.
   - `contractor_id` (UUID) — FK на `contractors`.
   - **RLS:** ✅ Включён (Изоляция через `company_id`).
2. **`contract_files`**: Документы договора.
   - **RLS:** ✅ Включён.

### Индексы и функции
- Индексы по `company_id`, `contractor_id`, `object_id`.
- RPC `calculate_contract_works`: расчет выработки.

## 🧠 Бизнес-логика
- **Контроль сроков:** В `ContractTableView` реализована логика расчета оставшихся дней:
  - `> 30 дней`: серый текст.
  - `0 - 30 дней`: янтарный текст (Expiring soon).
  - `< 0 дней`: красный текст (Overdue).
- **Финансы:** Автоматический расчет НДС (включен/сверху) на основе `vat_rate` и `is_vat_included`.
- **Изоляция:** Multi-tenancy обеспечивается через принудительную фильтрацию по `company_id` во всех Repository методах.

## 🔌 Интеграции
- **Edge Functions:** `ks2_operations` (формирование актов).
- **Storage:** Bucket `contracts` для файлов.
