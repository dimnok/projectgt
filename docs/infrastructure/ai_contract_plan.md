# 🤖 Интеграция ИИ Агента "План по договору"

**Дата:** 12 мая 2026  
**Статус:** ✅ Реализовано

---

## 🎯 Общее описание

Мы добавили функционал умного анализа договоров с помощью ИИ. Агент берет активный договор компании, считает его сумму и дату сдачи, сопоставляет это с объемом уже выполненных работ и рассчитывает план на каждый оставшийся день, включая план по выработке (в рублях) и количеству монтажников. В конце ИИ выдает короткую человечную рекомендацию (например, "Отстаем от графика, необходимо вывести еще 2 человека").

---

## 🏗 Архитектура

Функционал разделен на два слоя:
1. **Edge Function (`analyze-contract-plan`)** - агрегирует данные из БД и общается с LLM (OpenRouter / gpt-4o-mini).
2. **Flutter App** - содержит Entity (модель), Riverpod Provider для кэширования и вызова API, и визуальный виджет для главного экрана.

### 1. Backend: Edge Function (`supabase/functions/analyze-contract-plan/index.ts`)

**Алгоритм работы:**
1. Получает `company_id` (и опционально `contract_id`) в теле POST-запроса от клиента.
2. Если `contract_id` не передан, запрашивает в таблице `contracts` самый свежий активный договор по этой компании (`status = 'active'`).
3. Берет параметры договора: `amount` (сумма), `end_date` (дата сдачи) и `object_id`.
4. Делает вызов SQL RPC-функции `calculate_contract_works` для получения точной суммы выполненных работ по сметам именно этого договора.
5. Делает запрос в таблицу `works`, чтобы собрать статистику закрытых смен на объекте этого договора **за последние 7 дней**. Высчитывается "Средний факт за 7 дней": скорость выработки в день и среднее число монтажников на объекте.
6. Вычисляется дефицит: `Необходимое число монтажников` минус `Среднее число за последние дни`.
7. Отправляет эти цифры в виде строгого системного промпта через OpenRouter API модели `openai/gpt-4o-mini`. ИИ не придумывает цифры, а лишь формирует читаемую рекомендацию на основе рассчитанного дефицита.
8. Возвращает клиенту JSON:
```json
{
  "contract_number": "Д-123",
  "contract_amount": 10000000,
  "executed_amount": 3500000,
  "days_left": 45,
  "required_installers": 10,
  "daily_plan_amount": 150000,
  "installers_plan_today": 10,
  "recommendation": "Темп работ недостаточен. Чтобы успеть в срок, срочно выведите дополнительно 3 человек."
}
```

### 2. Frontend: Слой данных и бизнес-логики

- **Модель:** `lib/features/home/domain/entities/ai_contract_plan.dart`  
Описывает `freezed`-сущность `AiContractPlan` с необходимыми полями.
- **Provider:** `lib/features/home/presentation/providers/ai_contract_plan_provider.dart`  
Экспонирует `FutureProvider`, который достает `activeCompanyId`, вызывает Supabase RPC-функцию `analyze-contract-plan` и парсит JSON в модель.

### 3. Frontend: Слой UI

- **Виджет:** `lib/features/home/presentation/widgets/ai_contract_plan_widget.dart`  
Отображает рекомендации ИИ, план по монтажникам и ежедневную норму выработки.
- **Интеграция:** Виджет добавлен на оба дашборда (`HomeMobileDashboard` и `HomeDesktopDashboard`), рядом с виджетом `ContractProgressWidget`.

## 🛠 Как внести изменения в работу агента

1. **Если нужно изменить норму выработки (15000 руб):** 
Зайдите в `supabase/functions/analyze-contract-plan/index.ts`, найдите строку `Норма выработки на 1 монтажника: 15000 руб/смена` в переменной `systemPrompt` и измените её.

2. **Если нужно привязать к конкретному договору (а не последнему активному):**
В Riverpod-провайдере `aiContractPlanProvider` добавьте передачу параметра `contract_id` в тело запроса.

3. **Если нужно использовать другую ИИ-модель:**
В `index.ts` функции, в параметрах `body` для OpenRouter измените `"model": "openai/gpt-4o-mini"` на нужную (например, `anthropic/claude-3.5-sonnet`).

## 💡 Полезные советы по отладке

- Проверять логи Edge Function можно в интерфейсе Supabase -> Edge Functions -> analyze-contract-plan -> Logs.
- При изменении Dart модели не забудьте выполнить `flutter pub run build_runner build -d`.
- Edge Function деплоится командой: `supabase functions deploy analyze-contract-plan`. (Должны быть заданы ENV переменные `OPENROUTER_API_KEY`).
