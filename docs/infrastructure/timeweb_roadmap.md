## 🚀 2. Проект StroikaPRO: Структура на сервере

Проект развернут в изолированной директории `/root/StroikaPRO/`. Эта инсталляция полностью отделена от других проектов (MiniApp).

### 📁 Файловая иерархия
```text
/root/StroikaPRO/
├── docker/                 # Управление контейнерами
│   ├── .env                # Переменные окружения и секреты
│   ├── docker-compose.yml  # Манифест всех сервисов (13 штук)
│   ├── reset.sh            # Скрипт полной очистки данных
│   └── volumes/            # Постоянные данные (Mounts)
│       ├── api/            # Конфиг Kong (gateway)
│       ├── db/             # Данные и скрипты PostgreSQL
│       ├── storage/        # Файловое хранилище (S3-compatible)
│       ├── functions/      # Скомпилированные Edge Functions
│       └── logs/           # Системные логи Logflare/Vector
├── supabase/               # Конфигурация Supabase CLI
│   ├── migrations/         # SQL-миграции структуры таблиц
│   ├── functions/          # Исходный код Edge Functions
│   └── config.toml         # Настройки CLI
└── package.json            # Описание монорепозитория
```

## ⚙️ 4. Технические детали реализации

### Изоляция сервисов через Tailscale
В файле `docker-compose.yml` критические сервисы привязаны исключительно к внутреннему IP Tailscale. Это предотвращает доступ извне.
```yaml
# Реализованная настройка в StroikaPRO/docker/docker-compose.yml
services:
  studio:
    ports:
      - "100.107.237.40:3000:3000" # Панель управления
  kong:
    ports:
      - "147.45.99.66:8000:8000"   # Публичный API
      - "100.107.237.40:8000:8000" # Приватный API (MCP)
  db:
    ports:
      - "100.107.237.40:5432:5432" # Прямой доступ к БД
```

### База данных (PostgreSQL)
Используется образ `supabase/postgres:15.8.1`.
- **Схема:** `public` (основная), `auth` (пользователи).
- **Тестовые таблицы:** `test_table_self_hosted`, `test_mcp_success` (созданы для проверки MCP).

## 🤖 6. Настройка MCP для Cursor

Для того чтобы Cursor мог анализировать вашу базу данных, в `.cursor/mcp.json` добавлены следующие конфигурации:

1.  **`supabase-self-hosted`**: Работает через Kong (порт 8000). Позволяет использовать стандартные инструменты Supabase (список таблиц, выполнение SQL).
2.  **`stroikapro-db`**: Прямое подключение к Postgres (порт 5432). Самый мощный вариант для сложного анализа данных.
